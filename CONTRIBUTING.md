# Contributing to CV Playground

Thank you for your interest in contributing to CV Playground! This document provides guidelines for contributing to the project.

## How to Contribute

### Reporting Issues

- Check if the issue already exists
- Provide a clear description of the problem
- Include steps to reproduce the issue
- Mention your environment (OS, Python version, GPU/CPU)

### Suggesting Enhancements

- Describe the enhancement in detail
- Explain why it would be useful
- Provide examples if possible

### Adding New Experiments

If you want to add a new CV experiment:

1. **Fork the repository**
2. **Create a new branch**
   ```bash
   git checkout -b feature/your-experiment-name
   ```

3. **Add your experiment**
   - Keep code clean and well-commented
   - Include visualizations where appropriate
   - Add markdown cells explaining the technique
   - Test your code thoroughly

4. **Update documentation**
   - Add a section in README.md describing your experiment
   - Update requirements.txt if you add new dependencies

5. **Submit a Pull Request**
   - Provide a clear description of your changes
   - Reference any related issues

## Code Style

- Follow PEP 8 guidelines for Python code
- Use clear, descriptive variable names
- Add comments for complex operations
- Include docstrings for functions

## Notebook Guidelines

- Clear markdown explanations before code cells
- Visualize results whenever possible
- Keep outputs in the notebook for easy viewing
- Use consistent formatting throughout

## Ideas for Contributions

- Add new CV tasks (object detection, depth estimation, etc.)
- Improve visualizations
- Optimize model performance
- Add interactive demos
- Improve documentation
- Fix bugs or issues

## Questions?

Feel free to open an issue for questions or discussions about the project.

Thank you for contributing!
